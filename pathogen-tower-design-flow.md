# PATHOGEN TOWER — 설계 Flow (통합본)

> 감염·면역 학습용 몬스터 등반 RPG.
> 병원체를 포획해 파티를 짜고, 인체를 10층 파고들어 면역 사령부를 무너뜨린다.
> **암기 대상(면역학)이 곧 게임 규칙(상성)이 되도록** 설계한다.

---

## 0. 최상위 원칙 — 엔진 고정, 데이터만 성장

이 프로젝트의 뼈대는 단 하나다.

```
엔진(거의 안 바뀜)         데이터(무한히 늘어남)
──────────────            ──────────────────
전투 계산                  MOVES         공격기 사전
3층 상성 판정              ABILITIES     방어특성 사전
효과 원시 처리 + 턴 틱      EFFECTIVENESS 상성표
포획 / 층 / 보스 관리 ←읽음  MONSTERS      몬스터(위 id만 참조)
                          BOSSES
```

**몬스터 추가 = `MONSTERS` 배열에 객체 하나.** 기술·상성·판정 로직은 전부 사전과 엔진에 이미 있으므로 코드는 손대지 않는다. 엔진을 건드리는 유일한 경우는 "세상에 없던 새 메커니즘(효과 원시)"이 등장할 때뿐이다.

## 1. 상성 시스템 — 포켓몬과 다른 "비대칭" 구조

포켓몬은 같은 18타입끼리 물고 무는 **대칭 정사각형**(불→풀→물→불)이다.
이 게임은 **공격축과 방어축이 서로 다른 목록**인 **비대칭 직사각형**이다.

- **공격 타입 = 때리는 방식** (면역이 병원체를 공격하는 수단, 병원체가 숙주를 공격하는 수단)
- **방어 특성 = 버티는/숨는 방식** (병원체의 은신·회피)

"포식이 포식을 때린다" 같은 무의미한 칸이 없고, 대신 **"어떤 면역이 어떤 은신처를 뚫나"**가 그대로 표가 된다 → 표 한 줄이 곧 면역학 암기 카드.

### 피해배수 = 3층 곱셈

```
최종배수 = ①타입상성  ×  ②태그방어  ×  (③효과 필드/무적)
```

| 층 | 무엇 | 데이터 위치 |
|---|---|---|
| ① 타입상성 | 공격타입 × (방어특성 또는 태그값) | `EFFECTIVENESS[type][key]` |
| ② 태그방어 | 방어자 특성이 공격자 "태그"에 반응 | 특성의 `resistTag` |
| ③ 상태효과 | 무적·피해감소 필드 등 | 전투 중 `fx[]` |

정의 안 된 칸은 자동 **×1(보통)**. 그래서 표를 다 안 채워도 안 깨진다. (타입상성은 과증폭 방지를 위해 0~3배로 clamp.)

## 2. 공격 타입 목록

### 병원체 계열 (내 병원체가 사용)
| 타입 | 뜻 | 특이 상성 |
|---|---|---|
| 세포용해 lysis | 막천공·직접파괴 | — |
| 효소독소 toxin | AB형 저격독소 | 물리장벽 ×1.4 |
| 초항원 superantigen | 면역 폭주 유발 | — |
| 조직융해 spread | 확산효소 | **물리장벽·바이오필름 ×2** |
| 내독소 endotoxin | LPS/LTA 염증 | 보체순환 ×1.4 |
| 단백질오폴딩 misfold | 프리온 전용 | 특수(개종) |

### 면역 계열 (보스가 사용)
| 타입 | 굉장(×2 대상) | 무효(×0 대상) |
|---|---|---|
| 포식소화 phago | 무방비 | 피막·세포내·항산성·바이오필름 |
| 옵소닌표적 opsonin | **피막** | 항체무력화·세포내 |
| 항체중화 antibody | 세포외 | 항체무력화·항원변이·세포내 |
| 활성산소 oxidative | — | 활성산소중화 |
| 보체MAC complement | **그람음성** | 보체회피 |
| NET포박 net | — | 세포내 |
| 세포독성 ctl | **세포내** | 피막·바이오필름(세포외 반감) |
| 대식세포각성 th1 | **세포내·항산성** | — |
| 인터페론 interferon | **피막바이러스** | 피막 |

핵심 교훈: 세포외 = 항체/옵소닌/보체, 세포내 = CTL/Th1, 결핵(항산성) = Th1, 바이러스 = 인터페론.

## 3. 방어 특성 목록 (패시브)

병원체측: 무방비 · 피막장벽 · 활성산소중화 · 항체무력화 · 보체회피 · 항산성막 · 바이오필름 · 항원변이 · 포자내성 · **무핵산(프리온: 전 면역 ×0)**

인체(보스)측: 물리장벽 · 보체순환 · **점액섬모(마스크)** · **라이소자임**
→ 마스크/라이소자임은 타입이 아니라 **공격자 태그**를 본다(아래 §4).

## 4. 태그 축 — 인체 방어가 "무엇을 보고" 반응하는가

방어특성이 병원체의 방어타입이 아니라 **속성 태그**를 보는 경우가 있다. 라이소자임은 세포벽을, 마스크는 침입경로를 본다. 그래서 몬스터는 태그 묶음을 든다.

| 태그 축 | 값 | 이 태그를 읽는 인체 방어 |
|---|---|---|
| **pathway** 경로 | 호흡기·소화기·혈액매개·상처·피부·점막·접촉 | 점액섬모→호흡기 ½ |
| **wall** 세포벽·외피 | 그람+·그람−·항산성·피막바이러스·진균벽·원충 | 라이소자임→그람+ ½, 보체→그람− ×2, 인터페론→피막바이러스 |
| **location** 위치 | 세포외·세포내 | 항체→세포외, CTL/Th1→세포내 |
| (예정) size | 미시·대형(연충) | 호산구·IgE→대형 |
| (예정) tropism | 신경친화·간친화 | 면역특권·BBB |

특성의 조건부 방어는 특성이 자기 안에 들고 있다:
```js
mask:     { name:'점액섬모',  resistTag:{ pathway:{ respiratory:0.5 } } }
lysozyme: { name:'라이소자임', resistTag:{ wall:{ gram_positive:0.5 } } }
```
→ 새 특성 추가가 자기완결적. 엔진은 곱셈 한 번 더 하는 게 전부.

## 5. 효과 원시(primitive) + 턴 틱 — 전용기의 기반

모든 기술(기본기·전용기)은 `power / type / acc` + **효과 리스트 `effects[]`**. 기본기는 리스트가 짧고, 전용기는 원시들을 조합한 것뿐. **별도 "전용기 엔진"은 없다.**

엔진이 가진 고정 원시 라이브러리:

| 원시 kind | 효과 | 파라미터 |
|---|---|---|
| `buff` | 스탯 증감(±%) | stat, pct, turns, target |
| `status` | 상태이상 부여 | status(confusion/stun), chance |
| `invuln` | 무적(잠복) | turns |
| `field` | 진영 피해배수 | side:'incoming', factor |
| `dot` | 지속피해 | power, turns |
| `convert` | 개종(프리온) — 성장형 지속피해 | power |
| `heal` | 회복 | pct |

`stacks:n` 규칙: buff는 pct×n, invuln은 turns×n, field는 n번 곱, dot는 power×n.
**턴 틱**: 매 라운드 끝에 dot/convert 피해 적용 → 모든 fx의 turns 감소 → 만료 제거.

### 전용기 = 원시 조합 예시 (전부 데이터, 엔진 무수정)
```js
// 헤르페스 신경절 잠복 = 2턴 무적을 2스택 → 4턴
ganglion: { type:'misfold', power:0, effects:[ {kind:'invuln', turns:2, stacks:2, target:'self'} ] }

// 콜레라 범람 = 받는 피해 0.5배 필드 2스택 → 0.25배
flood:    { type:'toxin', power:8, effects:[ {kind:'field', side:'incoming', factor:0.5, turns:5, stacks:2, target:'self'} ] }

// 황색포도알균 TSST = (적 공격력+50% 버프) → (적 혼란) 순차
tsst:     { type:'superantigen', power:20,
            effects:[ {kind:'buff', stat:'attack', pct:50, turns:3, target:'enemy'},
                      {kind:'status', status:'confusion', chance:1, target:'enemy'} ] }
```
> 엔진을 건드리는 유일한 순간: **없던 원시(kind)가 필요할 때.**
> 예) 프리온 「템플릿 폴딩」의 개종 → `kind:'convert'`를 한 번 추가. 이후 그 원시를 쓰는 모든 전용기는 다시 데이터로만 조합된다.

## 6. learnset ↔ 전투 4슬롯

타입은 분류(class), 기술은 개체(instance). 한 타입 안에 기술이 여럿일 수 있다(불꽃 안에 화염방사·불대문자…). 그래서 몬스터는:

- `learnset` = **배울 수 있는 전체 풀** (개수 무제한)
- `signature` = **전용기**(선택)
- 전투 시작 시 자동으로 **4슬롯** 구성(전용기 우선 + learnset 순서로 채움)

같은 균이라도 딜형/탱커형으로 빌드가 갈린다(예: 황색포도알균 = 알파독소+PVL+히알루로니다제+장독소 vs 코아굴라제+PVL+알파독소+장독소).

## 7. 데이터 스키마 (그대로 복붙 = Phaser `src/data/`)

```js
// monsters.js
{ id:'staph', name:'황색포도알균', category:'세균', emoji:'🟡',
  tags:{ pathway:'skin', wall:'gram_positive', location:'extracellular' },
  maxHp:46, attack:11, defense:4, speed:8, captureRate:.42,
  ability:'proteinA',
  learnset:['alpha_toxin','pvl','hyaluronidase','coagulase','enterotoxin'],
  signature:'tsst' }

// moves.js
{ tsst:{ name:'TSST 폭주', type:'superantigen', power:20, acc:.8, sig:true, effects:[…] } }

// abilities.js
{ capsule:{ name:'피막장벽' },  mask:{ name:'점액섬모', resistTag:{pathway:{respiratory:0.5}} } }

// effectiveness.js  (키 = 방어특성 id 또는 태그값; 없으면 ×1)
{ opsonin:{ capsule:2, proteinA:0 },  complement:{ gram_negative:2, comp_evade:0 } }

// bosses.js  (방어특성/기술을 런타임에 랜덤 장착)
{ id:'immune_hq', name:'면역 사령부', maxHp:230, attack:15, defense:6,
  abilityPool:['barrier','comp_patrol','mask','lysozyme'],
  movePool:['m_phago','m_opsonin','m_ctl','m_th1', …] }
```

### 몬스터 추가 체크리스트
1. `MONSTERS`에 객체 추가 (기존 기술·특성·태그만 쓰면 여기서 끝).
2. 새 기술이 필요하면 `MOVES`에 추가 (원시 조합).
3. 새 특성이 필요하면 `ABILITIES`에 추가, 상성이 필요하면 `EFFECTIVENESS`에 칸 추가.
4. 없던 메커니즘이면 그때만 엔진에 원시 `kind` 1회 추가.

## 8. 보스 설계

- 10층 단일 관문 「면역 사령부」.
- 매 전투마다 **방어특성을 랜덤 장착**(물리장벽/보체순환/마스크/라이소자임) + 면역 공격기 풀에서 **랜덤 4기술**.
- 그래서 "내 파티가 어디서 강하고 어디서 막히는가"를 알아야 공략 가능 → 다양한 병원체를 모으게 강제.
- 예: 보스가 옵소닌표적을 쓰는데 내 병원체가 피막이면 크게 맞음 / 보스가 마스크면 호흡기 병원체 딜 반감 → 다른 균 투입.

## 부록 — 우리가 확정한 설계 판단들

- 공격/방어는 **다른 목록**(비대칭 상성). 완전 대칭 포켓몬식은 생물학이 깨져서 비채택.
- `location`은 특성이 아니라 **태그**로 승격(항체/CTL/Th1 상성이 깔끔).
- 전용기는 **원시 조합**으로, 새 엔진 없이 데이터화. 새 메커니즘일 때만 엔진 1회 확장.
- 방어는 기술이 아니라 **특성(패시브)**. 판정은 항상 *한쪽 공격기 vs 반대쪽 방어특성/태그* 단일 축.
- 1~9층은 수집·육성, 10층 보스가 상성 두뇌전의 중심(난이도 곡선).
