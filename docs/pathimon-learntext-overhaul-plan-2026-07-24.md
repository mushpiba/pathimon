# learn text 정비 플랜 (2026-07-24)

학습 텍스트를 맥락 연결 기반으로 바꾸고, 남는 포인트는 학습모드 전용으로 표출한다.
분석: 학습포인트가 전투 중 무작위(`randomLearningPoint`)로 튀어 맥락이 없고, 상호참조가 단독 표출 시 깨진다.

## 확정 설계

- **맥락 연결이 기본.** 플레이어가 기술을 쓰면 그 기술에 묶인 학습포인트를 표출(무작위 대체).
  근거 데이터: 노트 `작업메모`의 `기술↔학습포인트 대응`(81/82 노트 보유). 예: `탄저 독소 → L4·L5·L6`.
- **남은 포인트 → 학습모드 전용.** 어느 기술에도 안 묶인 포인트(순수 감별점·역학).
  - (a) 층 클리어/승리 화면(학습모드): leftover 1개
  - (b) 도감 "학습" 탭: 해당 패시몬 전체 포인트 열람
  - (c) 포획 시: 도전모드 1개(기존) / 학습모드 2개
- **흡수성 재작성**(콘텐츠, 점진): 자기완결 · 한 항목=한 개념 · 후크/대조/왜. `WORKFLOW.md`/`TEMPLATE`에 원칙 반영.
- **자기완결(A)**: 완료 — 요로콜리 L5, 페스틱 L7 dangling 수정.

## 구현 단계 (코드 먼저)

- **1. 파서 매핑** — `작업메모`의 `기술↔학습포인트 대응` 파싱 → 기술명→L번호→profileMemo 인덱스.
  MonsterData/RuntimeMonster에 `movePointMap: Record<MoveId, number[]>` 추가. 테스트.
- **2. 맥락 표출** — `game/learning.ts`에 `contextualLearningPoint(monster, moveId)`(매핑 우선, 없으면 무작위 폴백) +
  `leftoverLearningPoints(monster)`. `battle/turn.ts`가 기술 사용 시 이걸로 표출. 테스트.
- **3. 학습모드 표출** — (a) 층 클리어 leftover, (c) 포획 학습모드 2개. (b) 도감 학습 탭(UI). 테스트.
- **4. 콘텐츠** — 흡수성 재작성(점진) + 규격 반영. 별도.

각 단계 테스트 게이트.
