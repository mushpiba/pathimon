# Pathimon Phaser/Vite 설계

## 목표

Pathimon은 병원체와 면역학 학습 규칙을 전투 규칙으로 녹인 등반형 몬스터 RPG다. 첫 Phaser/Vite 단계는 최종 디자인 완성이 아니라, 전체 플레이 골격과 학습 규칙이 실제로 돌아가는지 확인하는 것이다.

첫 마일스톤에 포함할 것:

- 타이틀 화면
- 짧은 스토리/컷씬 화면
- 1층 전투 진입
- 4개 기술 선택
- 전투 중 캡슐 포획 버튼
- 기술 사용 전 기술 설명 UI
- 피해, 상성, 효과, 전투 로그 표시
- 승리 후 재화 획득
- 전투 후 상점 화면
- 다음 층 진행

기존 `pathogen-tower.html`은 규칙 검증용 원형으로 남긴다. 새 Phaser/Vite 앱은 이후 확장과 APK 포장의 기준 코드가 된다.

## 선택한 기술 스택

- Vite
- Phaser
- TypeScript
- 우선 정적 웹 배포
- APK는 이후 Capacitor 같은 웹-앱 포장 도구로 진행

TypeScript를 선택한 이유는 이 프로젝트가 데이터 중심으로 커지기 때문이다. 패시몬, 기술, 특성, 태그, 보스, 효과 원시의 모양을 타입으로 고정해두면 로스터가 늘어날 때 데이터 실수를 빨리 잡을 수 있다.

## 구조

규칙과 화면을 분리한다.

`src/data/`

- `monsters.ts`: 패시몬 로스터
- `moves.ts`: 기술 정의
- `abilities.ts`: 방어 특성과 태그 반응
- `effectiveness.ts`: 공격 타입 대 특성/태그 상성표
- `bosses.ts`: 보스 템플릿과 기술 풀
- `labels.ts`: 한국어 표시명

`src/battle/`

- `loadout.ts`: 전용기와 learnset을 전투 4슬롯으로 구성
- `effectiveness.ts`: 3층 피해배수 계산
- `effects.ts`: 효과 원시와 라운드 종료 틱
- `damage.ts`: 피해 계산
- `capture.ts`: 캡슐 포획률 계산
- `turn.ts`: 플레이어 행동, 적 행동, 기절, 승패 전환

`src/state/`

- `runState.ts`: 층수, 파티, 활성 패시몬, 재화, 캡슐, 상점 상태
- `factory.ts`: 데이터 템플릿에서 런타임 인스턴스 생성

`src/scenes/`

- `BootScene`: 게임 시작과 공통 스타일/리소스 준비
- `TitleScene`: 시작 화면
- `StoryScene`: 초반 스토리/컷씬 화면
- `BattleScene`: 필드, 유닛, 기술 선택, 캡슐, 로그, 설명 패널
- `ShopScene`: 전투 후 상점
- `BossIntroScene`: 10층마다 보스 등장 연출

`src/ui/`

- HP 바
- 전투 메뉴
- 기술 버튼
- 기술 설명 패널
- 캡슐/행동 버튼
- 로그 패널
- 상점 아이템 버튼

## 전투 화면 요구사항

전투 화면은 Pokerogue식으로 읽기 쉬운 배치를 유지한다.

- 적 패시몬은 우측/상단에 배치
- 플레이어 패시몬은 좌측/하단에 배치
- 이름(표시명과 학명), HP 바(숫자 없음), 방어 특성, 상태/효과 칩 표시
- 하단 명령 영역에 4개 기술 슬롯 표시
- 전투 중 별도 캡슐 버튼 표시
- 기술을 확정하기 전에 설명을 읽을 수 있는 UI 제공
- 전투 로그는 계속 읽을 수 있어야 함
- 레벨 표시는 사용하지 않음

캡슐 버튼은 숨겨진 보조 메뉴가 아니라 기술과 같은 단계의 전투 행동이다. 일반 전투에서는 포획을 시도하고, 보스전에서는 비활성화하거나 보스 포획 불가 상태를 명확히 보여준다.

기술 설명 UI는 기술 선택 영역에서 바로 접근한다. 첫 구현에서는 마우스 hover, 키보드 focus, 터치 선택 중 하나 이상으로 설명 패널을 갱신하면 된다.

설명 패널에 표시할 정보:

- 기술 이름
- 공격 타입
- 위력
- 명중률
- 효과 요약
- 어떤 면역 방어/태그와 상호작용하는지 짧은 학습 설명

## 핵심 루프

1. `TitleScene`에서 시작한다.
2. `StoryScene`에서 짧은 스토리 패널을 보여준다.
3. 1층 `BattleScene`으로 들어간다.
4. 플레이어는 4개 기술 중 하나를 쓰거나 캡슐을 사용한다.
5. 적이 살아 있고 포획되지 않았다면 적 턴을 진행한다.
6. 라운드 종료 효과를 처리한다.
7. 적을 쓰러뜨리거나 포획하면 재화를 지급하고 `ShopScene`으로 이동한다.
8. 상점은 첫 구현에서 다음 행동을 제공한다.
   - 패시몬 각성: 전용기 추가 또는 해금
   - 패시몬 진화: 랜덤 방어 특성 부여
   - 캡슐 구입
   - 다음 층 이동
9. 다음 층으로 진행한다.
10. 10층마다 `BossIntroScene`을 거쳐 보스전에 들어간다.

## 학습 모델

학습 내용은 별도 퀴즈가 아니라 전투 규칙 안에 둔다.

- 공격 타입은 병원체 또는 면역이 공격하는 방식이다.
- 방어 특성은 대상이 버티거나 숨거나 노출되는 방식이다.
- 태그는 침입 경로, 세포벽/외피, 위치를 설명한다.
- 피해배수는 다음을 곱한다.
  - 공격 타입 상성
  - 태그 기반 방어 반응
  - 현재 효과 필드 또는 잠복/무적

전투 피드백은 왜 효과가 있었는지 설명한다.

- "옵소닌표적 -> 피막장벽: 효과가 굉장했다"
- "점액섬모가 호흡기 태그를 반감했다"
- "세포내 위치라 항체중화가 닿지 않았다"

## 첫 마일스톤의 시각 방향

첫 Phaser 버전은 최종 아트보다 가독성과 동작 확인을 우선한다.

- 초반에는 단순 도형, 이모지, 임시 스프라이트를 사용한다.
- 데스크톱과 모바일에서 UI 크기가 흔들리지 않게 한다.
- 루프가 재미있어지기 전까지 장식 요소를 늘리지 않는다.
- 첨부 레퍼런스처럼 전투 흐름이 한눈에 보이는 구도를 유지한다.

## 검증 기준

첫 마일스톤은 다음이 실제로 동작하면 완료다.

- Vite 개발 서버가 실행된다.
- Phaser 게임 화면이 표시된다.
- 전투 화면에 이름(표시명과 학명), 숫자 없는 HP 바, 방어 특성, 상태/효과가 표시된다.
- 전투 화면에 레벨 숫자는 표시되지 않는다.
- 타이틀에서 스토리, 전투로 전환된다.
- 4개 기술 버튼이 데이터에서 렌더링된다.
- 기술 설명 패널이 기술 사용 전에 갱신된다.
- 일반 전투에서 캡슐 버튼으로 포획을 시도할 수 있다.
- 보스전 규칙에서는 캡슐 포획이 불가능하다.
- 최소 1개 기술이 데이터 기반 피해배수로 적에게 피해를 준다.
- 승리 시 재화를 얻고 상점으로 이동한다.
- 상점에서 다음 층으로 이동할 수 있다.

## 이번 단계에서 제외

다음은 첫 마일스톤 범위가 아니다.

- 최종 픽셀 아트
- 사운드
- 전체 스토리 작성
- 전체 상점 밸런싱
- 4슬롯 수동 편성 UI
- 저장/불러오기
- APK 빌드
- Godot 지원
