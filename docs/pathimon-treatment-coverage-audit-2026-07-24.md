# 처치기(적 기술) 커버리지 감사 — 80종

_생성일 2026-07-24 · **데이터 수정·검증 완료**(vitest `dataIntegrity` 커버리지 가드 통과). 이 표는 수정 후 최종 상태다._

## 0. 이 문서가 답하는 것

적(보스·트레이너 = 인체 방어·치료 행위)이 각 패시몬을 때릴 때 배율은 노트 `대처법`의 처치가 적 기술에 **등록돼 있느냐**로 정해진다.
- **×4 직접처치**: 그 병원체를 실제로 **죽이는 약물·백신·면역**이 기술 `targetTags`에 걸릴 때 (`battle/bossMatchup.ts` `bossMoveEffectiveness`)
- **×2 간접처치**: **물리제거·수술 + 증상약**(해열제·진통제·수액 등)이 걸릴 때
- **×1**: 아무것도 안 걸림 (예방·위생 포함)

현재 적 기술은 **19개**(`data/bossAttackMatchups.ts` `BOSS_ATTACK_MOVE_IDS`). 매칭은 정밀(약물명 단위)이라 계열 블랭킷이 아니다 — 하지만 그래서 **노트가 쓴 실제 약물명이 기술에 없으면 ×4가 영영 안 뜬다.** 이 문서는 그 구멍을 종별로 뽑은 것이다.

> **계열은 매칭이 아니라 분류에 쓴다.** 노트 `대처법`의 `계열:` 값으로 각 처치를 배율 범주로 나눈다:
> - **×4 직접** 🟥직접(항균제·항바이러스제·구충제·항진균제·항원충제·항독소) + 🟦면역(선천/세포성/**백신·항체**)
> - **×2 간접** 🟪물리(물리제거·수술) + 🟨지지(대증요법: 해열제·진통제·지사제·수액 등)
> - **무관** ⬜예방(위생·멸균·가열 = 감염 전 예방, 배틀 배율 없음)

## 1. 요약 (사용자 #2 재정의 반영: 물리제거·수술은 ×2)

| 상태 | 종수 | 뜻 |
|---|---|---|
| ✅ ok | 76 | 등록된 ×4 약물이 하나라도 있음 |
| 🔧 fixable | 0 | ×4 약물이 노트엔 있는데 기술에 미등록 → 등록하면 살아남 |
| 🩹 x2only | 4 | ×4 약물 자체가 없음(수술·대증만) → **정상**. ×2가 답인 종 |

- 등록하면 ×4가 살아나는 미등록 약물: **0종** (§B).
- 🩹 x2only는 버그가 아니다: 아니사키(내시경 적출만)·스파르강(수술만)·시가콜리(EHEC—항생제 금기, MATCHUP §5-4)처럼 **약이 없는 게 학습 내용**이다. 이 종들은 ×2(물리/지지)가 반드시 작동해야 한다.

## 2. 확정된 설계 (사용자 결정 A~D 반영)

- **배율 재정의**: ×4 = 병원체를 죽이는 약물·백신·면역 / ×2 = 물리제거·수술 + 증상약(해열제·진통제·지사제·수액) / ×1 = 예방·위생.
- **매칭 정밀 유지**: 계열 블랭킷 안 함(아시클로버가 전 바이러스에 ×4 뜨는 문제 회피). ×4는 약물명 문자열 매칭.
- **(A) 백신 ×4 인정**: 백신도 병원체 직접 대응이라 학습상 ×4. 단 표시에 `(백신)` 명시.
- **(B) 물리제거 계열 매칭 허용**: 표기 난립(내시경 적출/외과적 제거/충체 적출...) 대신 `물리제거` 계열이면 ×2 성립. 개별 철자 등록 불필요.
- **(C) 진단검사·위생 항목**: 대처법에 우선 그대로 두되, 학습상 중요하면 UI `learnText`로 노출(별도 정리, ×4/×2 아님).
- **(D) 복합 약물 문구**: 파서가 약물 셀을 `,` `/` `·`(가운뎃점)으로 쪼개고 `N제` 접미사 제거 → "이소니아지드·리팜핀·에탐부톨·피라진아마이드 4제"가 개별 약물로 분해돼 매칭(이 표는 이미 그렇게 계산됨). 남은 철자 "피라진아마이드"→"피라진아미드"만 노트에서 수정.
- **보스=트레이너**: speed는 죽은 스탯(패시몬 항상 선공)이라 무시. 차이는 **HP + 방어특성**만. 트레이너 = 보스처럼 19풀 랜덤 4개.
