# Pathimon Notes

패시몬을 새로 추가할 때 사용하는 원본 노트 폴더입니다.

## 입력 흐름

1. 사용자가 이 양식으로 패시몬 노트를 추가한다.
2. 이름이 비어 있으면 노트 내용을 바탕으로 이름 후보 7개를 추천한다.
   - 이름 후보는 `drafts/NAME_PROMPT.md`의 가중치와 형식을 따른다.
   - `a`~`e`는 특징을 비교적 직접 드러내는 후보, `f`~`g`는 더 포켓몬스럽고 익살스러운 후보로 둔다.
3. 이미지가 없으면 기존 패시몬 이미지 방식에 맞춰 캐릭터풍, 실사풍 이미지를 각각 준비한다.
   - 먼저 `app/public/images/pathimon`과 `app/dist/images/pathimon`에 현재 생성되어 있는 모든 패시몬 이미지를 확인한다.
   - 새 이미지가 기존 전체 이미지 풀과 실루엣, 색 조합, 장식 요소, 구도 면에서 너무 유사하게 보이지 않는지 확인한다.
   - 새 이미지는 그 병원체의 대표 특징을 한눈에 알아볼 수 있는 시각 키워드 1개를 중심으로 잡는다.
   - 특정 기존 패시몬 이름을 나열하기보다, 피해야 할 모양/색/구도 유형과 새 병원체의 고유 특징을 적는다.
4. 확정된 패시몬은 야생 조우와 스타터 후보에 등장할 수 있도록 데이터에 추가한다.
   - 1차 확정 후보는 `drafts/NAME_SELECTIONS.json`의 선택 순서를 기준으로 자동 등록된다.
   - 노트 파일의 `이름:` 필드와 `drafts/NAME_SELECTIONS.md/json`을 함께 갱신한다.
   - `app/src/data/pathimonNoteData.ts`는 선택표의 노트 경로를 읽어 `buildPathimonFromNote`로 자동 변환한다.
   - 새 기술 ID는 수동 매핑이 없으면 `패시몬ID_move_번호` 형식으로 자동 생성된다.
   - 활성 패시몬 로스터는 패시몬 노트에서 생성된 `NOTE_MONSTERS`만 사용한다.
   - 최신 노트 기반 패시몬은 `starterCandidateRoster()`와 `wildEncounterRoster()`에서 우선 노출된다.
5. 기존에 없는 기술은 기술 목록에 추가한다.

## 인게임 등장 체크

- 새 패시몬 노트를 최종 후보로 쓰려면 `drafts/NAME_SELECTIONS.json`에 번호, 파일 경로, 확정 이름을 넣는다.
- `NOTE_MONSTERS`는 `drafts/NAME_SELECTIONS.json`의 오래된 순서에서 최신 순서로 자동 생성된다. 최신 항목일수록 스타터 후보와 초반 야생 조우에서 먼저 보이도록 처리한다.
- 기존 자산 ID를 유지해야 하는 패시몬은 `pathimonNoteData.ts`의 `NOTE_OPTION_OVERRIDES`에만 예외를 둔다.
- 스타터 화면은 기본 스타터 1마리 뒤에 최신 노트 기반 패시몬을 우선 배치한다.
- 야생 조우는 최신 노트 기반 패시몬을 먼저 배치한다.
- 새 패시몬을 추가한 뒤에는 `dataIntegrity.test.ts`, `runState.test.ts`, `starterSelectUi.test.ts`의 로스터 회귀 테스트로 인게임 노출을 확인한다.

## 이미지 생성 규칙

- 캐릭터풍은 기존 Pathimon 캐릭터 자산의 톤을 유지한다.
  - 192x192 투명 PNG 전투 스프라이트로 저장한다.
  - 굵은 외곽선, 픽셀풍 고광택, 읽기 쉬운 실루엣, 귀엽지만 위협적인 표정을 유지한다.
  - 기존 패시몬과 같은 주 실루엣을 반복하지 않는다.
- 실사풍은 기존 Pathimon 현미경/실사 자산의 톤을 유지한다.
  - 384x384 이미지로 저장한다.
  - 학습용 현미경 사진 또는 비그래픽 교육용 실사풍 이미지를 우선한다.
  - 실사풍에는 캐릭터풍의 상징 모티프를 억지로 섞지 않는다.
  - 병원체의 실제 현미경 형태, 염색 양상, 세포 내 위치, 관찰 가능한 병변 중 하나를 기준으로 삼는다.
  - 병변 이미지는 실제로 그 병원체를 설명하는 데 필요한 경우에만 쓰고, 현미경 이미지가 더 자연스러우면 현미경 형태를 우선한다.
- 새 패시몬 이미지를 만들기 전에는 현재 생성되어 있는 모든 기존 패시몬 이미지를 비교 대상으로 삼는다.
- `기존과 겹치지 않게 할 요소`에는 특정 패시몬명 대신 전체 이미지 풀과 겹치지 않아야 할 기준을 적는다.
- 프롬프트에는 반드시 `기존과 겹치지 않게 할 요소`와 `반드시 반영할 병원체 특징`을 함께 적는다.

## 기술 슬롯 규칙

- 노트에는 `슬롯`을 적지 않는다.
- `준비기`는 항상 1번 슬롯에 들어간다.
- `공격기`는 공격기 목록에서 2번, 3번 슬롯에 랜덤으로 들어간다.
- 공격기가 1개뿐이면 공격기 슬롯은 1칸만 채운다.
- `전용기`는 항상 4번 슬롯에 들어간다.
- 학습모드에서는 전용기가 기본 해금 상태다.
- 도전모드에서는 전용기가 개체별로 잠겨 있다.
- 도전모드에서 상점 아이템 `이상한 사탕`을 사용하면 선택한 개체 1마리의 전용기만 해금된다.
- 도전모드에서 상점 아이템 `진화의 돌`을 사용하면 진화 가능한 개체 1마리를 진화시킨다.
- 잠긴 전용기는 4번 슬롯에 보이되 사슬 X 표시로 막는다.

## 기술 문구 규칙

- `effect`는 나중에 기술 이펙트와 효과음을 연결할 때 사용할 연출 키 이름이다.
- `effect`는 사용자가 구체적으로 지정하기 전까지 공란으로 둔다.
- 기술 바로 아래의 `description`은 기술창과 도감에 보이는 짧은 기본 설명이다.
- `결과` 안의 `description`은 실제 결과가 선택되었을 때 전투 로그에 보이는 문장이다.
- 기술 바로 아래의 `learnText`는 기술창과 도감에 보이는 기본 학습 문구다.
- `결과` 안의 `learnText`는 학습모드에서 실제 결과가 선택되었을 때 추가로 출력되는 학습 문구다.
- 한 기술 안에 확률별 결과가 여러 개 있으면 `결과` 목록으로 나눈다.
- 한 기술이 1단계, 2단계, 3단계처럼 순환하면 `결과` 대신 `단계` 목록으로 나눈다.
- 단계별 기술은 기술의 `위력`에 `단계별(0/40/200)`처럼 요약하고, 실제 위력 숫자는 각 `단계` 안에도 적는다.
- `description`은 항상 `learnText` 바로 위에 적는다.
- 공격력 랭크업, 무적 같은 전투 수치 변화는 텍스트 로그 대신 UI 아이콘으로 표시할 예정이다.
- 수치 변화 자체는 `효과`에 적고, `description`에는 전투에서 벌어진 사건을 적는다.
- 상태이상은 `STATUS_CONDITIONS.md`에 있는 공식 목록만 우선 사용한다.

## 메모 규칙

- `메모:`는 `방어특성:` 바로 아래에 둔다.
- 메모는 인게임 스테이터스 프로필 창에 그대로 표시된다.
- 기본 분량은 4줄로 둔다.
- 1줄째는 병원체 타입과 방어특성, 2줄째는 구조/위치, 3줄째는 감염경로, 4줄째는 대표 특징이나 병인 키워드를 정리한다.
- 후보 번호, 출처 강의, 검수 필요 같은 작업용 내용은 `작업메모:`에 둔다.

## 상태 아이콘 예정

- 공격력 랭크업: 빨간색 칼 아이콘과 위쪽 화살표.
- 무적: 반이 황금색인 방패 아이콘.

## 능력치 규칙

- 속도 능력치는 새 노트에 적지 않는다.
- 전투 순서는 항상 패시몬이 선공하는 것으로 처리한다.
