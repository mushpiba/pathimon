# 패시몬 이름 생성 프롬프트

이 문서는 새 패시몬 노트가 들어왔을 때 이름 후보를 만드는 기준이다.
최근 1차 후보 59개 선택 결과를 반영해 가중치를 정했다.

## 선택 결과 요약

사용자 선택값은 `1=a`, `2=b`, `3=c`, `4=d`, `5=e`, `6=f`, `7=g`로 해석했다.

| 후보군 | 선택 수 | 비율 | 해석 |
|---|---:|---:|---|
| a | 34 | 57.6% | 가장 직관적인 핵심 특징 이름 선호 |
| b | 8 | 13.6% | 병변/질환/생활사 이미지가 보이는 이름 선호 |
| c | 4 | 6.8% | 형태나 기전이 드러나는 이름 일부 선호 |
| d | 2 | 3.4% | 학명 기반 말장난은 낮은 비중 |
| e | 3 | 5.1% | 대표 키워드 압축형은 보조 후보 |
| f | 3 | 5.1% | 익살형은 일부만 채택 |
| g | 5 | 8.5% | 포켓몬스러운 반복음/의성어 후보는 보조로 유효 |

## 가중치

이름 후보 7개를 만들 때 다음 비율을 기본값으로 둔다.

| 후보 위치 | 역할 | 가중치 |
|---|---|---:|
| a | 병원체의 가장 큰 특징을 바로 보여주는 대표 후보 | 35% |
| b | 병변, 전파 경로, 숙주 위치를 살린 후보 | 18% |
| c | 독소, 효소, 구조, 생활사 같은 기전을 살린 후보 | 14% |
| d | 학명 일부를 자연스럽게 섞은 후보 | 8% |
| e | 짧고 강한 키워드 압축 후보 | 10% |
| f | 포켓몬스럽고 익살스러운 후보 1 | 7% |
| g | 포켓몬스럽고 익살스러운 후보 2 | 8% |

## 프롬프트

```text
패시몬 노트를 읽고 한국어 패시몬 이름 후보 7개를 만든다.

목표:
- 이름만 봐도 병원체의 대표 특징, 병변, 전파 경로, 독소/구조/생활사 중 하나가 떠오르게 한다.
- 너무 학술적인 직역보다 포켓몬 이름처럼 짧고 기억하기 쉬운 조어를 우선한다.
- 기존 패시몬 이름과 너무 비슷한 음절, 실루엣, 인상을 피한다.
- 이미 확정된 패시몬 이름 목록과 겹치지 않는다.

후보 구성:
- a: 가장 직관적인 대표 특징 이름.
- b: 질환/병변/전파 경로가 보이는 이름.
- c: 독소, 효소, 구조, 생활사 등 기전이 보이는 이름.
- d: 학명 또는 한국어 병원체명 일부를 자연스럽게 섞은 이름.
- e: 짧고 강한 키워드 압축형 이름.
- f: 더 포켓몬스럽고 익살스러운 이름.
- g: 반복음, 의성어, 말장난을 조금 더 살린 포켓몬풍 이름.

선호:
- 2~5음절 안팎의 짧은 이름을 우선한다.
- 병원체 고유 특징이 없는 일반적인 이름은 피한다.
- 귀엽거나 익살스러워도 병원체 특징을 완전히 잃으면 탈락시킨다.
- f/g 후보는 장난스럽되 너무 유치하거나 뜻을 알 수 없는 이름은 피한다.

출력:
| 후보 | 이름 | 의도 |
|---|---|---|
| a | ... | ... |
| b | ... | ... |
| c | ... | ... |
| d | ... | ... |
| e | ... | ... |
| f | ... | ... |
| g | ... | ... |
```

## 선택 결과 반영 원칙

- a 후보처럼 직관적인 이름이 가장 자주 선택되었으므로, 첫 후보는 항상 가장 안정적인 대표 이름으로 둔다.
- f/g 후보는 반드시 만들되, 최종 선택 비율은 낮았으므로 전체 후보의 분위기를 지나치게 장난스럽게 끌고 가지 않는다.
- 학명 말장난만으로 만든 이름은 선택률이 낮으므로 d 후보에만 제한적으로 둔다.
- 이름 선택 후에는 `NAME_SELECTIONS.md`와 노트의 `이름:` 필드를 함께 갱신한다.
