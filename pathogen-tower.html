엔진 고정 · 데이터만 추가

몬스터 추가 = MONSTERS에 객체 하나. 아래 3층 상성과 효과 원시는 이미 엔진에 있으므로 코드는 손대지 않는다.

몬스터 스키마

{ id, name, category, emoji,
 tags:{pathway,wall,location}, // 인체 방어가 읽는 축
 maxHp, attack, defense, speed, captureRate,
 ability:'proteinA', // 방어특성 id
 learnset:['alpha_toxin',...],// 배울 수 있는 전체 풀
 signature:'tsst' } // 전용기(선택) }

전투엔 learnset에서 자동으로 4개만 장착(전용기 우선). 배우는 수는 무제한, 들고 나가는 건 4.

피해배수 = 3층 곱

① 타입상성 EFFECTIVENESS[공격타입][방어특성·태그]
② 태그방어 특성의 resistTag가 공격자 태그에 반응 (마스크→호흡기 ½, 라이소자임→그람양성 ½)
정의 안 된 칸은 자동 ×1(보통).

전용기 = 원시효과 조합

모든 기술은 power/type + effects[]. 전용기도 새 엔진 없이 원시 조합:
· 신경절잠복 = invuln ×2스택
· 범람 = field 0.5 ×2스택(→0.25)
· TSST = buff+status 순차
원시(buff·status·invuln·field·dot·convert)에 없는 새 메커니즘일 때만 엔진 1회 추가.
